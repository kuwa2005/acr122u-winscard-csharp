## 依存関係

1. 実装班は #13 の差し戻しを解消し、新規 `Acr122uCardInspector` を `.sln` に追加する。
2. UXレビュー班と CI整備班は、`--help` / `--version` / `--json` / 不正オプションが実装された後に実機不要チェックを具体化する。
3. 実機テスト計画班は、Phase 1 再実装後にカード種別別の手動確認を実行または未実施理由を記録する。
4. コードレビュー班は、再実装差分が出てからレビューを開始する。
5. セキュリティレビュー班は、JSON / Markdown / trace の出力仕様が固まった時点で機密値と安全境界を確認する。
6. ドキュメント班は、テスト班、品質チェック班、セキュリティレビュー班の結果が揃ってから最終反映する。
7. リリース班は、CI、コードレビュー、品質チェック、ドキュメント整備が完了してから成果物作成へ進む。

## 最新工程順序

直近の最優先は #13 の Phase 1 実装差し戻し解消である。以降は次の順で進める。

1. #13 実装班: 新規 `Acr122uCardInspector`、責務分離、正式 CLI、JSON、read-only default、非対話実行を実装する。
2. #22 コードレビュー班: #13 の実装 commit/push 後に、Critical / High / Medium の severity 付きでレビューする。
3. #14 品質チェック班: レビュー後に Release build、CLI、JSON、trace、エラー、非対話、生成物除外を再確認する。
4. #12 ドキュメント班: #13 / #14 / #22 の結果が揃ってから README、仕様、設計、テスト結果を最終反映する。
5. #17 CI整備班: Phase 1 実装後に GitHub Actions の Release build と実機不要チェックを整備する。
6. #18 リリース班: 署名、配布、バージョン、チェックサム、リリースノート、納品物一覧を確定する。
7. 総括 / プロジェクトリーダー: Issue、ドキュメント、CI、成果物の整合を確認し、納品可能判定を出す。

並行可能な作業は #19 セキュリティレビュー、#20 実機テスト計画、#21 UXレビューである。ただし、最終判定は #13 の実装成果を確認してから行う。

## 納品完了条件

- `dotnet build .\NFC_CardReader.sln --configuration Release` が成功している。
- `Acr122uCardInspector` の `--help`、`--version`、`--json <path>`、`--trace`、不正オプション、非対話実行が期待どおりに終了する。
- 既定動作が read-only default で、reader 設定変更、書き込み系 APDU、鍵探索、FeliCa サービス総当たりを行わない。
- ATR、UID、ATS、リーダー概要、PICC parameter、取得失敗、安全方針で未取得、対象外がコンソールと JSON に矛盾なく出る。
- 実機テスト計画に基づき、保有カードで確認できた範囲と未保有/未実施の範囲が明記されている。
- コードレビュー、品質チェック、セキュリティレビューで未解決の Critical / High がない。
- README、設計書、機能仕様、テスト/品質/レビュー結果、リリース手順が最新状態を反映している。
- `bin/`、`obj/`、`logs/`、実機ログ、UID を含む未マスク資料、署名用秘密情報がコミットや配布資料に混入していない。
- リリース班が納品物一覧、バージョン、チェックサム、署名/未署名の扱い、GitHub Releases 手順を確定している。

## 現時点の抜け漏れと対応

- CI/CD は未割当だったため #17 を追加した。
- リリース/配布は #1 に含まれていたが独立追跡が弱いため #18 を追加した。
- セキュリティ/法務/倫理は設計に記載済みだが独立レビューがなかったため #19 を追加した。
- 実機テストは報告項目として存在するが、カード種別別計画が未分離だったため #20 を追加した。
- CLI UX と非対話実行は品質レポートで要修正だが、利用者体験としての担当が未分離だったため #21 を追加した。
- コードレビューは想定役割にあるが実装完了後の受け皿がなかったため #22 を追加した。
