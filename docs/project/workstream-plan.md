## 依存関係

1. 実装班は #13 の差し戻しを解消し、新規 `Acr122uCardInspector` を `.sln` に追加する。
2. UXレビュー班と CI整備班は、`--help` / `--version` / `--json` / 不正オプションが実装された後に実機不要チェックを具体化する。
3. 実機テスト計画班は、Phase 1 再実装後にカード種別別の手動確認を実行または未実施理由を記録する。
4. コードレビュー班は、再実装差分が出てからレビューを開始する。
5. セキュリティレビュー班は、JSON / Markdown / trace の出力仕様が固まった時点で機密値と安全境界を確認する。
6. ドキュメント班は、テスト班、品質チェック班、セキュリティレビュー班の結果が揃ってから最終反映する。
7. リリース班は、CI、コードレビュー、品質チェック、ドキュメント整備が完了してから成果物作成へ進む。

## 現時点の抜け漏れと対応

- CI/CD は未割当だったため #17 を追加した。
- リリース/配布は #1 に含まれていたが独立追跡が弱いため #18 を追加した。
- セキュリティ/法務/倫理は設計に記載済みだが独立レビューがなかったため #19 を追加した。
- 実機テストは報告項目として存在するが、カード種別別計画が未分離だったため #20 を追加した。
- CLI UX と非対話実行は品質レポートで要修正だが、利用者体験としての担当が未分離だったため #21 を追加した。
- コードレビューは想定役割にあるが実装完了後の受け皿がなかったため #22 を追加した。
